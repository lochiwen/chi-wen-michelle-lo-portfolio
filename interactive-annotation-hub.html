Project deep dive

Interactive Annotation Hub

A Replit prototype for digital note-taking in language learning, designed to show how students could highlight readings, create sticky annotations, organize notes, and collaborate inside a digital course reading workflow.

Open live demo

Project summary

This prototype began from a workplace need at Berlitz: a supervisor wanted to explore how language students could take notes digitally while reading course content. The goal was to make the desired LMS functionality visible, concrete, and easier for a platform partner to understand. An LMS, or Learning Management System, is software used to deliver courses and learning activities.

I designed the feature concept and built the prototype in Replit as a reference for the LMS company working with Berlitz. The prototype demonstrates the kinds of annotation, organization, collaboration, persistence, and export features Berlitz hoped could be considered for Parta.io, the LMS platform Berlitz was using with a partner company.

Product context

Audience
Language learners, instructors, and LMS stakeholders who need better reading-based note-taking workflows.
Core problem
Students need a way to connect notes directly to reading passages, organize them, and return to them later.
Design medium
Interactive Replit prototype used to communicate desired LMS functionality to a platform partner.
My contribution
Feature ideation, annotation UX design, workflow definition, prototype building, and stakeholder-facing product framing.

Prototype feature set

The prototype models a full annotation workspace, from creating notes inside a document to searching, filtering, exporting, and collaborating around those notes.

  1. Sticky-note annotations Students can click anywhere on the document to drop a sticky note when Annotation On mode is active.
  2. Text-linked highlights Students can highlight text to create an annotation tied to a specific passage, with highlights linked to notes.
  3. Rich note editing Notes support bold, italic, underline, bulleted lists, numbered lists, tables, images, hyperlinks, and mentions.
  4. Comment threads Annotations can hold comment threads, allowing students or collaborators to respond to a note.
  5. Eight-color system Highlights and sticky notes use yellow, orange, red, pink, purple, indigo, blue, and green for visual organization.
  6. Synced color changes Changing a highlight color updates its linked sticky note, and changing a sticky note updates the linked highlight.
  7. Tags and repository search Students can tag notes and search by content, tags, colors, and metadata from the repository panel.
  8. Filtered exports Exports can respect active filters, so learners download only the subset of notes they are currently reviewing.
  9. Repository panel A central panel shows all annotations and highlights, updates counts in real time, and lets users jump to any note.
  10. Interactive note behavior Notes expand on hover, edit inline on click, drag to new positions, and can be shown all at once or only on hover.
  11. Contextual controls Highlights support right-click context menus, hover tooltips, quick color changes, and delete actions.
  12. Persistence and history Content, positions, colors, tags, and annotation history auto-save to LocalStorage and persist across refreshes.
  13. Export and sharing paths Annotations can be downloaded as PDF, exported with the document, or shared to Gmail, Google Drive, or Notion.
  14. Multi-user collaboration The concept includes multi-user viewing, commenting, and @username mentions to tag collaborators.

Learning design rationale

For language learners, useful notes are not only private reminders. They are comprehension anchors tied to vocabulary, examples, grammar, cultural meaning, questions, and feedback. The prototype keeps notes close to the exact text that triggered them.

  • Highlights connect thinking to the source passage instead of separating notes from context.
  • Color categories let learners create personal systems for vocabulary, grammar, questions, and review.
  • The repository supports later study by turning scattered notes into a searchable learning record.

Stakeholder value

The prototype made an abstract LMS feature request tangible. Instead of only describing "digital notes," it gave the LMS partner a working interaction model for what Berlitz wanted students to be able to do.

  • Clarified feature expectations for Parta.io discussions.
  • Showed how reading, highlighting, note organization, and collaboration could live in one workflow.
  • Made product conversations easier by giving stakeholders something concrete to inspect and react to.

Interaction model

The design gives learners multiple ways to move between the reading surface and their study system.

Create in context

Click in the document or highlight a passage to create a note exactly where the learning moment happens.

Organize visually

Use synced colors and tags to group annotations by meaning, priority, question type, or review category.

Return quickly

Search, filter, repository jumps, animated scrolling, and counts help learners revisit important notes.

Discuss together

Comment threads, mentions, and multi-user support turn annotations into a collaborative learning space.
