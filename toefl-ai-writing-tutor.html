Project deep dive

TOEFL AI Writing Tutor

A working Custom GPT for TOEFL writing practice at Berlitz, designed to give students structured prompts, timed practice, rubric-based feedback, and strict guardrails against answer-writing shortcuts.

View full prompt PDF

Project summary

This project began from a workplace request at Berlitz: a supervisor wanted a ChatGPT-based TOEFL AI Writing Tutor that students could use for practice. I researched TOEFL writing task formats, timing, response expectations, and rubric-based feedback needs, then designed and built the Custom GPT. The prompt in the Create tab defines the assistant's role, practice flow, task rules, timing behavior, and feedback boundaries.

The design focuses on disciplined practice rather than shortcut help. The tutor must not write answers for students, teach before submission, provide sample answers too early, give numerical scores, or predict TOEFL proficiency levels.

Learning context

TOEFL learners who need guided writing practice with clear task boundaries and feedback.

Practice tasks

The tutor supports three TOEFL writing practice formats. Each format has its own structure, so students can practice the selected task without mixed instructions.

  1. Build a Sentence Generates three scrambled-word items, each with one question and 7-10 words that allow one grammatically correct solution.
  2. Write an Email Presents a scenario, exactly three bullet points, a To line, and a Subject line without extra advice.
  3. Academic Discussion Includes a professor's question, Student A response, Student B response, and a final prompt.

Practice rules and TOEFL-aligned guidance

Beyond the three task types, I designed rules and guidance to keep the experience close to TOEFL-style practice and prevent the AI from turning into an answer-writing helper.

  1. Task selection first The tutor must not present a task until the student chooses one of the three task types.
  2. One task at a time The assistant handles only one task at a time and must not mix rules or formats across tasks.
  3. Official time limits Automatically provides the correct Google timer link based on the selected task: 6 minutes for Task 1, 7 minutes for Task 2, and 10 minutes for Task 3.

Interaction flow

The tutor opens with a fixed message that sets expectations: students should finish each task in one sitting and avoid dictionaries, grammar tools, or translation tools.

  • The student chooses Build a Sentence, Write an Email, or Academic Discussion.
  • The tutor immediately presents the selected task, provides the timer sentence and the correctly timed link, then stops.
  • The student opens the timer link and presses Start, so the countdown begins only when they are ready to write.
  • The tutor waits for the student's submission before giving any teaching, feedback, or sample answer.
  • After submission, feedback follows a fixed order so the experience stays predictable.

AI guardrails

The prompt is intentionally strict because a helpful AI tutor can accidentally reduce practice quality by giving hints, examples, or polished answers before the student has attempted the task.

  • Must not write answers for the student.
  • Must not teach grammar, vocabulary, strategies, or examples before submission.
  • Must not give numerical scores or predict TOEFL levels.
  • Must not change task definitions, rubrics, timer wording, or task formats.

Feedback design

Feedback is allowed only after the student submits an answer. The sequence is designed to be useful without turning the assistant into a score predictor or answer generator.

Check constraints

For Academic Discussion, the tutor first checks whether the response is outside the recommended 100-140 word range.

Use rubric language

The tutor gives qualitative feedback based on the official rubric without assigning numerical scores.

Find patterns

The tutor identifies 2-4 recurring error patterns so students know what to improve next.

Model reachable writing

The revised sample answer should be slightly above the student's level, not an overly advanced native-level rewrite.

Prompt architecture

After researching the TOEFL writing formats and feedback expectations, I translated those requirements into mandatory rules, exact wording requirements, and prohibited behaviors so the AI would act like a controlled practice environment instead of an open-ended writing helper.

  • Fixed opening message establishes the practice rules before task selection.
  • Timer rule requires exact sentence wording and matching official time limits.
  • Task-specific rules define required components for each practice format.
  • Feedback rules reflect rubric-based qualitative review rather than score prediction.
  • Allowed behaviors are separated from prohibited behaviors to reduce unwanted tutoring.

Learning design rationale

The goal is to preserve productive struggle. Students still have to generate their own answers, but they receive structured feedback after they have completed a realistic practice attempt.

  • Timed practice supports test-like pacing and writing stamina.
  • Delayed feedback protects independent performance before instruction begins.
  • Error-pattern feedback helps students focus revision effort without overwhelming them.
  • Level-appropriate sample answers make improvement feel reachable.

Portfolio takeaway

TOEFL AI Writing Tutor shows my ability to design AI-supported learning experiences with clear pedagogical boundaries. Rather than using AI to replace student work, I used prompt design to protect practice integrity, structure feedback, and make the tutor safer for exam preparation.

  • Designed and built a Custom GPT for TOEFL writing practice.
  • Created strict AI behavior rules that prevent premature answers, examples, hints, and scoring claims.
  • Balanced learner support with realistic timed practice and rubric-based feedback.
